# Deployment Guide for Vercel

## Pre-Deployment Checklist

✅ **Build Test Passed** - Project builds successfully locally
✅ **Dependencies** - All required dependencies are in `package.json`
✅ **Next.js Config** - Configuration is set up correctly
✅ **Environment Variables** - None required for current setup

## Files to Deploy

The following files/folders should be included in your Git repository:

- `/src` - All source code
- `/public` - Static assets (images, fonts)
- `/fonts` - Axiforma font files (optional - if not committing, ensure they're on Vercel)
- `package.json` - Dependencies
- `package-lock.json` - Dependency lock file
- `next.config.js` - Next.js configuration
- `tailwind.config.ts` - Tailwind configuration
- `tsconfig.json` - TypeScript configuration
- `postcss.config.js` - PostCSS configuration
- `vercel.json` - Vercel configuration (optional)

## Excluded from Git

The following are in `.gitignore` and will NOT be deployed:

- `/node_modules` - Installed by Vercel
- `/.next` - Build output (generated by Vercel)
- `.env*.local` - Local environment variables
- `.vercel` - Vercel deployment config

## Deployment Steps

1. **Commit all changes to Git:**
   ```bash
   git add .
   git commit -m "Prepare for Vercel deployment"
   ```

2. **Push to GitHub/GitLab/Bitbucket:**
   ```bash
   git push origin main
   ```

3. **Connect to Vercel:**
   - Go to [vercel.com](https://vercel.com)
   - Click "New Project"
   - Import your Git repository
   - Vercel will auto-detect Next.js settings

4. **Deploy:**
   - Vercel will automatically build and deploy
   - Build command: `npm run build`
   - Output directory: `.next` (auto-detected)

## Important Notes

- **Font Files**: Make sure `public/fonts/Axiforma/*.ttf` files are committed to Git or uploaded to Vercel
- **Images**: All images in `/public` folder will be served statically
- **Environment Variables**: None required for current setup, but can be added in Vercel dashboard if needed
- **Build Settings**: Vercel auto-detects Next.js, but `vercel.json` is included for explicit config

## Post-Deployment

After deployment:
1. Test all pages on the live URL
2. Verify images load correctly
3. Check that fonts are loading
4. Test the booking form (currently client-side only)
